<task type="auto">
  <name>Task 1: Simplify Progress Model in Prisma</name>
  <files>prisma/schema.prisma</files>
  <action>
    Modify `prisma/schema.prisma` to implement a simplified `Progress` model for single-user (global) use.
    - Remove `userId` and its relationship to `User`.
    - Set `chapterId` as the primary key (`@id`).
    - Include `completedAt` (DateTime) to track completion.
    - Remove the existing `Progress` model and replace it with:
      ```prisma
      model Progress {
        chapterId   String   @id
        completedAt DateTime @default(now())
      }
      ```
  </action>
  <verify>
    <automated>npx prisma validate</automated>
  </verify>
  <done>Prisma schema is valid and contains the simplified Progress model.</done>
</task>

<task type="auto">
  <name>Task 2: Update Database Schema</name>
  <files>prisma/schema.prisma</files>
  <action>
    Apply the schema changes to the database.
    - Run `npx prisma db push --force-reset` to align the database with the new simplified model (force-reset is acceptable here as we are in early development and removing the User relation).
    - Alternatively, use `npx prisma migrate dev --name simplify_progress` if you prefer migrations.
  </action>
  <verify>
    <automated>npx prisma db pull</automated>
  </verify>
  <done>Database schema reflects the new Progress model.</done>
</task>

<task type="auto">
  <name>Task 3: Create Progress API Route</name>
  <files>src/app/api/progress/[chapterId]/route.ts</files>
  <action>
    Implement `src/app/api/progress/[chapterId]/route.ts` to handle progress state.
    - `GET`: Check if a record exists for the given `chapterId`. Return `{ completed: true }` if found, else `{ completed: false }`.
    - `POST`: Toggle the completion status. 
      - If it exists: Delete the record (unmark complete).
      - If it doesn't exist: Create the record (mark complete).
    - Ensure no authentication checks are performed (per D-PUBLIC-ACCESS).
  </action>
  <verify>
    <automated>curl -X GET http://localhost:3000/api/progress/test-id</automated>
  </verify>
  <done>API route successfully handles GET and POST for chapter progress.</done>
</task>
